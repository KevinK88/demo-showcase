Background

  • The project's goal is create a piece of software to help our client manage the development process for any games he may be developing.
  • The client is Adam C. Younis, an indie game developer from Australia. His website is upponhill.com
  • The main issue that the client currently faces is that the the time estimates are not very useful in the current software he uses.

Scope

  • The scope of our project is to implement a database for storing and loading any of the clients projects.
  • This will include allowing the client to manage previous projects as well as create new ones.
  • The client will also be able to create jobs for each project, and subsequently create a infinite depth of subtasks for those jobs.
  • The client will also be able to set, view, and edit various properties of the jobs, and subtasks, throughout his development process.
  • There will be a timed estimate given to each task marked as completed.
  • Additionally, the software is hosted on Heroku server.
  • Lastly, it will feature a Login and Sign up system

Proposed Solution

  • Let the client enter their own estimate, and then enter their own completion time, and determine the estimate completion time that way, and the client could set task priority manually.
  • This solution allows for the most freedom and input from the client when it came to task creation and task completion.
  • The client could also go inside of each task and subtask, respectively, and edit the estimates for each of those in the middle of their development process.
  • The accuracy would be determined based on several factors at this point and would show the data that determined the accuracy to the client.

Highlights

  • This project management software is unique in that it is made for one developer.
  • This allows the developer to closely monitor and set various properties of tasks themselves.
  • It is innovative in that the client gets to see how well they are at estimating tasks and prioritizing them.
  • It also shows the client data, that is often hidden from the user, which can also help with project organization.
